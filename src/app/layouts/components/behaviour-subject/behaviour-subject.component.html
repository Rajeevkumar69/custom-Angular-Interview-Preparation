Angular BehaviorSubject

BehaviorSubject is a special type of Observable in RxJS that holds a current value and use to share the data between the components in Real time. It's same as [(ngModel)] two way data binding technique.

Overview

A BehaviorSubject in Angular is a special type of Observable that holds and multicasts a current value to its subscribers. It must be initialized with a default value and immediately emits its last emitted value to any new subscriber.

Key Characteristics
  • Initial Value Required: Must be instantiated with an initial value, never undefined.
  • Emits Current Value Immediately: New subscribers instantly receive the latest value.
  • Multicast: Allows multiple observers to share the same data stream.
  • Access Current Value Directly: Use .value or .getValue() to get current value synchronously.
  • Hot Observable: Produces values as soon as created, regardless of subscribers.
Common Use Case

BehaviorSubject is commonly used for state management in Angular services, such as managing:

  • Current user's login status
  • Items in a shopping cart

Any new component that subscribes immediately gets the current state.

BehaviorSubject vs Subject
Feature BehaviorSubject Subject
Initial value Yes, required No initial value
Value on subscription New subscribers get the latest value immediately Subscribers only get values emitted after subscription
Use case Managing "value over time" or current state Event emitters (e.g., button click notification)
Best Practice

Expose BehaviorSubjects as public Observable in services to prevent external components from calling next() directly. This ensures controlled data flow and prevents unexpected state changes.

BehaviorSubject is powerful for managing shared state and reactive data in Angular applications.
