What are Angular Signals?

Angular Signals are a new reactive state management mechanism introduced to simplify change detection and improve performance. Signals represent reactive values that automatically track dependencies and update the UI when their value changes, without relying on Zone.js or complex RxJS streams.

Why Angular Introduced Signals
  • Simplifies state management compared to RxJS
  • Improves performance with fine-grained reactivity
  • Reduces unnecessary change detection cycles
  • Provides predictable and explicit data flow
Core Signal Types
  • Signal
    Holds a reactive value and notifies dependents when updated
  • Computed
    Derives values from other signals and recalculates automatically
  • Effect
    Runs side effects when dependent signal values change
Signals vs RxJS
  • Signals are synchronous and simpler to read
  • RxJS is better suited for async streams and complex flows
  • Signals reduce boilerplate and subscription management
  • Both can coexist in the same Angular application
Common Use Cases
  • Component-level state management
  • Optimized UI updates for forms and dashboards
  • Replacing simple RxJS subscriptions
  • Improving performance in large applications
Interview Focus: Signals provide fine-grained reactivity, reduce dependency on Zone.js, and simplify state handling compared to RxJS.
