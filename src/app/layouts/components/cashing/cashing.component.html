Angular Cashing

What is Caching?

Caching is a performance optimization technique where previously fetched or computed data is temporarily stored and reused instead of requesting it again. In Angular applications, caching reduces API calls, improves response time, lowers server load, and enhances user experience, especially in data-driven and large-scale applications.

Angular Caching Types
  • Build-Time Caching using Angular CLI to speed up rebuilds
  • Runtime Caching to reuse data during application execution
Runtime Caching (Client-Side)
  • HTTP Interceptors
    Intercepts HTTP requests and responses, stores GET responses in memory, serves cached data for repeated calls, and clears cache on data mutation requests.
  • RxJS shareReplay
    Shares a single HTTP response across multiple subscribers and prevents duplicate API calls.
  • Browser Storage
    Stores configuration data, preferences, or session-based values using localStorage or IndexedDB.
Advanced Angular Caching
  • Service Workers (PWA)
    Acts as a network layer, caches assets and API responses, enables offline access and faster load times.
  • SSR Transfer Cache
    Reuses server-side HTTP responses on the client to avoid duplicate API calls during hydration.
Interview Focus: Angular caching improves performance by minimizing redundant HTTP calls. HTTP Interceptors and RxJS are most commonly discussed in interviews.
