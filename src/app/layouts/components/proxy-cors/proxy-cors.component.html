Proxy & CORS in Angular

Proxy configuration helps Angular apps avoid CORS issues during development.

üåê What is an Angular Proxy?

An Angular proxy is a development-only configuration used with ng serve to redirect API calls from the frontend to a backend server.

It allows Angular to call backend APIs without triggering browser CORS errors.

üö´ Why CORS Error Happens

Browsers enforce the Same-Origin Policy.

  • Frontend: http://localhost:4200
  • Backend: http://localhost:8080

Since ports are different, the browser blocks the request.

üîÅ How Proxy Works (Flow)
  1. Angular app makes a request to /api/users
  2. Angular dev server intercepts the request
  3. Proxy forwards it to http://localhost:8080/api/users
  4. Backend responds to the proxy
  5. Proxy sends response back to Angular

The browser thinks it‚Äôs the same origin, so no CORS error occurs.

Create a file named proxy.conf.json


               {
               "/api": {
               "target": "http://localhost:8080",
               "secure": false,
               "changeOrigin": true,
               "logLevel": "debug"
               }
               }

Any request starting with /api will be redirected to the backend.

‚ñ∂Ô∏è How to Use Proxy

Add this to your angular.json or start command:


               ng serve --proxy-config proxy.conf.json

Now use relative URLs in your Angular services: 


               this.http.get("/api/users")
Important: Proxy works only in development. In production, CORS must be handled by the backend or web server (Nginx, Apache).
